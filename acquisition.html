Data Acquisition¶

Preparing the data involves the following one or two steps:

  1. Turning a Function Graph into a Vector Graph.
  2. Extracting feature points from a Vector Graph for matchline overlay comparison.

It can be applied to a lot of different applications:

  • Analyzing pen-strokes from a 2D input device.

    Partially working.

    Generic frontend for pen-strokes.

    Example: Recognising Asian characters. (See http://www.symboloh.com)

  • Analyzing a time-based series of Function Graph data.

    Not yet implemented :)

    Turns a Function Graph into a Vector Graph. Extracts feature points.

    Example: Frontend for non-realtime signal analysis.

    Example: Frontend for stock market quotes.

In different fields:

  • Pen-Stroke UI
  • Signal Analysis
  • Data Visualisation

Function Graph¶

Any time-based data set with an accurate sampling rate can be used to build a function f(x).

Extraction of Feature Points¶

The analyzer then converts the vector graph into a sequence of one Turn, one Corner and one Turn. The sequence - starting with a Turn - has one Corner terminated by a turn and is concatenated into Turn-Corner-Turn-Corner-Turn.

This input can be used to make a matchline overlay comparison (See Server) to previously recorded vector graphs.
